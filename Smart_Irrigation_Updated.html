Introduction

The AI-Based Smart Irrigation and Monitoring System presents an IoT-based intelligent irrigation and monitoring solution designed to enhance agricultural productivity. The system integrates Internet of Things (IoT) sensors and Artificial Intelligence (AI) to create an automated, data-driven field environment.


A Raspberry Pi 4 serves as the central edge controller, collecting data from sensors measuring soil moisture, temperature, humidity, and pH levels. A Convolutional Neural Network (CNN) model — trained via Jupyter Notebook — is incorporated for early detection of onion plant leaf diseases through manual image input and deep learning inference. All data is visualized via the ThingsBoard IoT cloud platform.

🌾 Automated Irrigation

Soil moisture-triggered pump activation — no manual intervention needed

🔬 AI Disease Detection

CNN with 96.09% accuracy detects 11 onion diseases from manually provided leaf images

🧪 pH & Soil Monitoring

Real-time pH and soil moisture data via MCP3008 ADC with analog sensor interfacing

📡 Cloud Monitoring

ThingsBoard dashboard for real-time remote monitoring and actuation

Problem Statement & Objectives

❌ Problem Statement

  • Manual irrigation wastes 40–60% of water due to over/under-watering
  • Plant disease detected late (lab-based) — crops already damaged by then
  • Farmers lack real-time field data for informed decisions
  • High operational cost due to manual labor in field management
  • No affordable expert advisory system for small and marginal farmers
  • Disease identification requires expensive lab tests — no affordable instant solution

✅ Objectives

  • Automate irrigation using real-time soil moisture and sensor data
  • Detect onion leaf diseases using CNN with ≥95% accuracy via Jupyter-trained model
  • Integrate multiple IoT sensors with ThingsBoard cloud platform
  • Provide GPT-4o powered expert agricultural advisory
  • Handle analog sensor data via MCP3008 ADC over SPI protocol
  • Build voice-first multilingual interface for low-literacy farmers

Proposed System

The proposed system is divided into two main modules:

Module 1: Hardware Control & Local GUI

  • Runs entirely on Raspberry Pi 4
  • Real-time sensor data collection via GPIO + MCP3008 ADC
  • CNN-based disease detection (Tkinter GUI) — manual image input
  • Auto actuation: pump via relay
  • Image preprocessing (OpenCV — Grayscale + Otsu)
  • ThingSpeak data upload for graphs

Module 2: Advanced Agentic AI (Cloud)

  • GPT-4o powered expert advisory agent
  • Google Search Tool for medicine identification
  • OpenAI Whisper (Speech-to-Text)
  • TTS Voice output in Marathi/Hindi/English
  • Conversational 24/7 agri-expert chat
  • Kept separate from Pi (resource optimization)
Why Separate? Raspberry Pi is an edge device with limited compute. GPT-4o requires high processing power and high-speed internet for API calls. Running it on Pi would cause system lag and could crash the irrigation control loop. This is a deliberate Resource Optimization decision.

Why CNN over SVM/Random Forest?

Automatic Feature Extraction — CNN learns features from images automatically. SVM/RF need manual feature engineering.

Spatial Hierarchy

Convolutional layers capture spatial relationships between pixels — crucial for identifying disease patterns in leaf images.

Why ReLU over Sigmoid/Tanh?

Solves Vanishing Gradient Problem. Simple calculation saves Pi CPU. Sparse activation ignores noise, keeps important features.

Dropout (0.5)

Randomly deactivates 50% of neurons during training. Prevents overfitting — model generalizes to unseen disease images.

MaxPooling 2×2

Downsampling reduces spatial dimensions. Keeps most important features. Reduces memory load on Raspberry Pi.

Optimizer: SGD (lr=0.01)

Stochastic Gradient Descent with learning rate 0.01. Achieves stable, smooth convergence over 50 epochs.

Data Preprocessing & Model Training (Jupyter Notebook)

  • CNN model was built and trained using a Jupyter Notebook for interactive experimentation and visualization
  • Initially, webcam was used for image capture — however, webcam image quality was insufficient for reliable CNN predictions
  • Decision taken: webcam removed; images are now manually provided directly to the model for disease identification — significantly improving accuracy
  • All images resized to 100×100 pixels with aspect ratio preservation (no distortion)
  • Black padding added to maintain original disease pattern proportions
  • Automatic dataset organization using os.walk — mirrors source folder structure
  • Only image files (.jpg, .png) processed; invalid files automatically filtered
  • High-quality JPEG output to preserve disease pattern details

Hardware Interface & Power Management

🏗️ Hardware Architecture Overview

The Raspberry Pi 4 Model B acts as the Master Controller for the entire system. It manages GPIO-based digital sensors, coordinates SPI communication with the ADC IC, runs the CNN inference engine, and handles cloud data publishing — all concurrently via Python multi-threading.

⚡ Why MCP3008? Raspberry Pi 4 does not have any built-in Analog-to-Digital Converter (ADC) pins. All its GPIO pins are purely digital. Sensors like the pH probe and Soil Moisture probe produce analog voltage signals (0–5V). To read these on the Pi, the MCP3008 ADC IC is used — it converts analog input into a 10-bit digital value (0–1023) and sends it to the Pi over the SPI protocol.

The SPI (Serial Peripheral Interface) bus uses four dedicated lines: MISO (Master In Slave Out — Pi reads data from ADC), MOSI (Master Out Slave In — Pi sends channel select to ADC), SCK (Serial Clock — Pi drives the clock), and CS/CE (Chip Select — Pi activates MCP3008). This full-duplex synchronous protocol ensures fast, reliable analog data transfer at up to 3.6 MHz.

⚡ Triple-Source Power Management

Critical design decision — three fully isolated power sources prevent electrical noise and protect all components.

🔋
Pi Power Rail

5V / 3A via USB-C
Source: Mobile Phone Adapter (5V/3A rated)
Powers: Raspberry Pi 4 + GPIO logic + DHT11

💧
Actuator Power Rail

12V / 2A DC Adapter
Dedicated solely to: Water Pump
Switched via: Relay module (opto-isolated)

⚗️
Sensor Power Rail

12V / 1A DC Adapter
Dedicated solely to: pH Signal Conditioning Module
Prevents pump noise from corrupting pH readings

🔗 Common Ground (GND) — Essential Design Principle

All three power sources share a Common Ground connection with the Raspberry Pi's GND pins. This establishes a single voltage reference point across the entire circuit. Without a common ground, analog sensor readings from the MCP3008 would be unreliable — voltage levels measured at the ADC would have no consistent reference, causing incorrect pH and moisture values. Common grounding is the foundation of stable, accurate multi-rail systems.
